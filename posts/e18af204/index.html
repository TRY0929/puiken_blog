flutterの単体テスト

1. 紹介

flutterのテストパッケージドキュメント：flutter-test

新しい機能を追加したり、既存の機能を変更した後、アプリが正常に動作することをどのように保証するのでしょうか？答えは、テストを書くことです！

単体テストを使用すると、関数やメソッド、クラスなど個々の動作を簡単に検証できます。testパッケージは単体テストを書くための基本フレームワークを提供し、flutter_testパッケージはウィジェットテストを行うための追加機能を提供します。

このチュートリアルでは、testパッケージの使い方をデモンストレーションします。内容は以下の通りです：

  1. testまたはflutter_testを依存関係に追加する
  2. テストファイルを作成する
  3. テスト対象のクラスを作成する
  4. 作成したクラスに対するテストを書く
  5. 複数のテストを1つのgroupに統合する
  6. テストを実行する

パッケージテストに関する詳細な情報は、上記testパッケージのドキュメントをご参照ください。

2. テスト依存関係の追加

DartパッケージがFlutterに依存していない場合、testパッケージを導入できます。testパッケージは、テストを書くために必要な基本的な機能を提供します。作成しているパッケージがWeb、サーバーサイド、そしてFlutterアプリで使われる場合、これが最適な方法です。

flutter pub addコマンドを実行して、testを依存関係に追加します。

3. テストファイルの作成

この例では、2つのファイルを作成します: counter.dart と counter_test.dart。

counter.dart ファイルは、lib フォルダ内にあるテスト対象のクラスを含み、counter_test.dart ファイルは test フォルダに配置され、テストそのものを含みます。

通常、テストファイルは Flutterアプリまたはパッケージのルートディレクトリ下にある test フォルダに配置します。テストファイルの命名は _test.dart とするのが一般的で、これは test runner がテストファイルを検索する際の慣例です。

7. テストの実行

これで Counter クラスとそのテストが準備できたので、テストを実行してみましょう！

IntelliJ または VSCode でテストを実行

IntelliJ と VSCode の Flutter プラグインはテストの実行をサポートしています。この方法でテストを実行するのが最適で、迅速なフィードバックループを提供し、ブレークポイントデバッグもサポートしています。

IntelliJ

  1. counter_test.dart ファイルを開きます。
  2. メニューから Run > Run 'tests in counter_test.dart' を選択します。または、キーボードショートカットを使ってテストを実行できます。

VSCode

  1. counter_test.dart ファイルを開きます。
  2. メニューから Run > Start Debugging を選択します。または、キーボードショートカットを使ってテストを実行できます。

ターミナルでテストを実行

ターミナルを開き、プロジェクトのルートディレクトリに移動して、以下のコマンドを入力することで、すべてのテストを実行することもできます。

以下のコマンドを実行して、指定したグループにあるすべてのテストを実行できます：
